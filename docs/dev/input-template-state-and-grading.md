# 输入型题卡状态与判题问题复盘

## 问题

- `input-cloze` 前面允许输入，但背面没有读取并判定输入结果。
- `input-cloze`、`input-exact` 这类输入型题卡会把上一次输入保留到下一次复习。

## 根因

- 前面把用户输入写入 `sessionStorage` 或回退缓存，但背面没有在读取后立即清理。
- `input-cloze` 只有前面输入逻辑，没有补完整的背面判题链路。

## 修复

- 所有输入型题卡在背面读取用户输入后立即清理该题缓存。
- `input-cloze` 参考 `input-exact/multi-line` 的做法，新增逐字符判题结果与逐空反馈。
- 输入型题卡的回退缓存统一改为按题目 key 隔离，避免不同题目互相污染。

## 预防规则

- 输入型题卡必须满足完整链路：前面采集输入，背面读取输入、给出判题反馈、立即清理缓存。
- 任何新加的输入型模板都必须回归验证“再次复习时输入框为空”和“背面能看到本次作答结果”。
